PDF → raw (PRN-style via QZ)

A real .prn file is whatever a specific Windows driver emits; this page cannot build that in the browser. Instead, QZ Tray can turn a PDF into ESC/POS, ZPL, or EPL and send that as a raw job (same pattern as the official demo’s “PDF” raw button).

When Printer language is ZPL, QZ raw PDF uses pageRanges: "1" in this demo (first page only for Path A). In-browser PDF→ZPL below can render all pages when All PDF pages is checked. EPL / ESC/POS are unchanged.

PDF → ZPL (in-browser)

Base pdf.js scale is DPI÷72 (PDF points → dots at that DPI). Leave Raster DPI empty to use 203. Optional pdf.js scale overrides that mapping entirely. Optional paper width and height (inches): when both are set, the raster is scaled to fit that dot box (paper × Raster DPI) using the fit policy below; leave both empty to use PDF size only. All PDF pages: one ^XA…^XZ per page. Preview = 1-bit after the same LUMA threshold as ^GFA. pageWidth / supersample = QZ raw PDF only.

WebUSB — same idea as ekartprintv3 (bulk OUT, chunked transferOut)

No QZ Tray: pair each printer once, then connect all remembered USB devices. With more than one open, pick the USB print target before sending ZPL. Requires Chrome or Edge on https:// or http://localhost (not file://). If claimInterface fails, another app or the OS may own the device — pause CUPS/close other drivers and retry.

Shown when at least one USB session is open. Add more devices with “pick device” — they stay connected together until you disconnect all.

On load, all remembered USB printers are opened when this is checked. QZ Tray jobs: if several system printers appear after Refresh printers, each print asks you to confirm the one selected in the printer dropdown.

TCP capture: Path A (QZ) vs Path B (browser)

Start python3 tools/print_capture_server.py first. It saves each job as .bin and (unless --no-post-process) also writes .png for ZPL with ^GFA or ^GFB, and (unless --no-ps-pdf) .pdf for PostScript via gs. Path A: QZ Tray converts PDF→ZPL internally, then sends raw bytes to host:port (bypasses CUPS PostScript). Path B: same as “browser” above — converts PDF→ZPL here, then one-click send to TCP.
